    # 序

    笔者曾负责国内某财产保险核心系统的研发与维护。工作内容涵盖了需求分析到开发测试的整个流程，对业务的理解要求非常高，但技术上相对老旧一些，对一名仍需要成长的程序猿来说帮助不是很大。笔者没有停止对技术的追逐，在第三年，选择了离开。虽然离职已有一段时日，但因为倾注了太多的心血，着实难以割舍。时至今日，依然会重新思考一些当时所遇到的问题。

    程序员都有一个重构的梦想，重构天、重构地、重构世间的空气，重构就好像是程序员实现自我价值的过程，好像是在打天下一样。笔者当年也曾给自己立下过FLAG，待学有所成之时，要以一己之力把收付费系统重构一遍。恰逢最近在学习微服务架构，于是又想到了这位曾经的“老朋友”，也想到了曾经的信誓旦旦。学有所成之日尚未到来，但躁动的心早已饥渴难耐。然而，在得不到原系统的代码、不能复用的情况下，纯手码出十余万行的规模，对于笔者来说，基本上是不可能完成的任务，毕竟IT从业者，个人时间非常有限，就算写个五年八年，最终憋出来了，但不知道哪年技术又革新了，这重构之路，永远都走不完的。

    所以，笔者在写这系列博客的时候，并非要做出什么惊天地泣鬼神的事情，而是以此为契机，对自己现有的技术栈做一个系统的梳理，对不太明确的地方有一个补充学习的过程。毕竟终归是要学习，留点笔记总是好的。

    于是，便有了这个系列的博客。以假定【重构保险的核心收付费系统】为背景，记录自己的学习过程，笔者技术粗浅，难免出现纰漏，如有发现还请及时指正，以免误导他人，在此谢过 🤝。

    本系列面向企业web全栈开发，对技术深度没有过多的要求，但技术广度会大一些：前端三大件、Vue全家桶、Spring全家桶、高可用、缓存技术、容器技术、运维基础都会有所涉猎，篇幅有限，这些东西想讲清楚真的是太多太多了，笔者尽自己所能阐述清楚，但如果遇到不太清楚的地方，还请读者自行学习。

    笔者业余时间有限，只是偶尔周末才有时间写一写，可能更新周期会比较长🤣......
