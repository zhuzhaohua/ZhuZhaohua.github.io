# 业务流程

  • 1.用户登录、选择自己权限所在的分公司，可以看到权限允许的功能菜单。
  • 2.首页上会显示相应的统计图以及当前的任务及任务状态。
  • 3.管理员权限可看到管理菜单，进行相关的设置。
  • 4.结算操作，以收保费为例：
    • 在保费结算操作界面，输入查询条件；
    • 得到待结算业务的查询清单；
    • 勾选业务，下一步；
    • 确认对象交易账号信息，我方交易账号信息，点击下一步，生成结算单；
    • 确认结算单，结算；
  • 5.可在任务管理中查看当前结算进度；
  • 6.可在凭证查询中查看结算凭证；
  • 7.可在综合查询中查询到结算的业务信息；

理论上一个用户只属于一个分公司，但一些权限高的用户（比如总公司用户），有权查看所有分公司业务。对于这些用户，需要给出一个选择所属公司的菜单。

前文提到收付费的各业务流程非常的相似，这给我们前端的开发提供很大的便利，对于【结算】这一类业务，它们的查询画面、清单画面、结算单画面、结算画面都非常相似，可以封装成组件，根据传入不同的参数，以调整其内容，这也是笔者最开始着手的事情——对结算功能进行抽象，并提取通用组件。

# 前端的多租户

在登录时，从后台拿到用户信息里的租户ID，即分公司的标识，如果是多个，给出选择画面；如果是一个，则默认进入。 我们要在所有发往后台的请求上，都添加租户ID，那么在登录得到ID后，要通过状态管理存起来，在请求拦截器中做统一的添加； 请求到达后台，在后台过滤器中做租户ID的过滤，没有则报错给前台，有的话，切换当前线程的数据源为当前租户。
